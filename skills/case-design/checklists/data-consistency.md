# 数据一致性 Checklist

生成数据一致性用例时，对照以下清单检查是否遗漏。

## 并发写入
- [ ] 多用户同时修改同一数据
- [ ] 同一用户多端同时操作
- [ ] 批量操作中部分失败

## 事务完整性
- [ ] 操作中断后数据状态（断网、崩溃、关闭页面）
- [ ] 部分成功的多步操作（如支付成功但订单创建失败）
- [ ] 回滚是否彻底

## 缓存与持久化
- [ ] 缓存与数据库数据不一致
- [ ] 本地存储与服务端数据不一致
- [ ] 刷新页面后数据是否保持

## 数据同步
- [ ] 多端数据同步延迟
- [ ] 离线操作后重新上线的数据合并
- [ ] 删除操作的级联影响
