# 安全性 Checklist

生成安全性用例时，对照以下清单检查是否遗漏。

## 注入攻击
- [ ] SQL 注入
- [ ] XSS（存储型、反射型、DOM 型）
- [ ] 命令注入
- [ ] 路径遍历（../../etc/passwd）
- [ ] LDAP 注入
- [ ] 模板注入

## 认证
- [ ] 暴力破解
- [ ] 凭证填充（使用泄露的密码库）
- [ ] 会话固定
- [ ] 会话劫持
- [ ] Token 过期后仍可使用
- [ ] 密码重置流程漏洞

## 授权
- [ ] 水平越权（访问同级别其他用户的数据）
- [ ] 垂直越权（普通用户访问管理员功能）
- [ ] IDOR（不安全的直接对象引用）
- [ ] 功能级别访问控制缺失

## 数据安全
- [ ] 敏感数据明文传输
- [ ] 敏感数据明文存储
- [ ] 日志中泄露敏感信息
- [ ] 错误信息泄露系统细节
- [ ] API 响应包含多余字段

## CSRF / CORS
- [ ] 跨站请求伪造
- [ ] CORS 配置过于宽松
- [ ] 缺少 CSRF Token
