# 状态转换 Checklist

生成状态转换用例时，对照以下清单检查是否遗漏。

## 基本覆盖
- [ ] 所有状态是否都已列出
- [ ] 每个状态的所有合法转换是否都已覆盖
- [ ] 每个状态的非法转换是否都已定义预期行为
- [ ] 初始状态是否明确
- [ ] 终态是否明确

## 异常转换
- [ ] 在任意状态下会话/连接断开
- [ ] 在任意状态下超时
- [ ] 在转换过程中被中断（如关闭浏览器）
- [ ] 重复触发同一转换
- [ ] 跳过中间状态直接到达后续状态（如篡改 URL）

## 并发状态
- [ ] 多端同时操作导致状态冲突
- [ ] 状态回退（如支付后退款）
- [ ] 状态过期（如订单超时自动取消）
